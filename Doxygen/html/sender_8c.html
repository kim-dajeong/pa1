Client side for a more reliable file transfer using UDP sockets.

Author

Bug

When sending two competing UDP protocols in a lossy/noisy channel, the second socket entering experiences delays and a greater timeout. Works fine for no loss.

Only accepts IPV4 Addresses, in whatever form they may be. Not meant for IPV6.

rsend() sends data reliably using UDP Sockets

Inputs: hostname, hostUDP port, filename, bytesToTransfer Outputs: Void

Sender Algorithm Skeleton:

  • Read from File (raw data).
  • Splice the file into sendable bits.
  • Create socket.
  • Send the file bits over through the socket.
  • Check for ack and increase index, repeat if nack received.
  • Terminate connection and close socket and file.
Parameters
hostnameThe hostname can be an IP Address or a fully-qualified name.
hostUDPportThe port which you are sending data over.
filenameThe a char pointer to the file you are reading from.
bytesToTransferThe number of bytes you want to read from filename.
